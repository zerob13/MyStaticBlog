技巧：如何让Ghost支持带日期的URL

从 Wordpress 迁移过来后，大部分问题都解决了，前两天甚至自己实现了下代码高亮，可惜拖慢网站速度，所以还是算了。不过最近遇到了一个新问题：

Google上搜到自己的网站的链接，大多数都还保留着原来 Wordpress 文章的 Link 格式 如下

http://www.zerob13.in/2012/12/09/gan-ku-di-lao-tie-jiang/

但是目前 Ghost 支持的格式如下：

http://www.zerob13.in/gan-ku-di-lao-tie-jiang/

这个问题让我一度很头疼，本来想看下 Ghost 的代码自己改下的，可惜看了5分钟后果断放弃了，代价太大，另外以后也不好维护。这时候我就想到了万能的 Nginx rewrite 大法。语法参考这里：HttpRewriteModule

反正大意就是通过 rewrite 匹配多出来的那串日期，然后 rewrite 去掉这个日期就好了。转换成代码就是如下的：

 rewrite ^/20../../../(.*)$ /$1 last;

这是最简单的一个配置，通过匹配我日期的最开头两个数字（因为我的博客都是20xx年以后的，所以不存在19xx的情况）以及后面的几个斜杠，然后替换成一个斜杠就好了。完整的 location 代码如下：

 location / {
                        rewrite ^/20../../../(.*)$ /$1 last;
                        proxy_pass http://127.0.0.1:2368/;
                        proxy_set_header Host $host;
                        proxy_buffering off;
                        }
